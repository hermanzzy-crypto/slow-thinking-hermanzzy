# 🏃 slow-thinking · 思考跑步机

> AI 让答案变得廉价，让"自己想"变得稀缺。
> 这个 skill 不替你思考，它逼你思考。

每天 5 分钟，AI 当陪练，不当替身。Claude Code 上的刻意思考训练 skill。

---

## ✨ 核心设计

**三段式流程**：

| 段 | 名称 | 时长 | 关键动作 |
|---|------|------|---------|
| 1 | 🎯 我是谁？ | 10s | 选择思考身份（个人/火山引擎/数智平台/自定义） |
| 2 | 📰 发生了什么？ | 60s | 三选一内容源（Top3 AI 新闻 / 用户链接 / 行业情报） |
| 3 | 💡 所以呢？ | 3-4 min | 抽思考卡片 → AI 对抗性追问 → 用户回答 |
| 4 | 💾 沉淀 | 自动 | Obsidian + 飞书双写 |

**铁律**：Step 3 中 AI 输出 ≤3 句，剩下全部用来**向用户提问**。
用户答 > AI 答。

## 🪪 身份系统

**预设身份**（直接用）：
- `volcano-engine.md` — 火山引擎员工视角
- `shuzhi-platform.md` — 数智平台 VeDI 视角

**动态生成**（其他身份）：
通过 `identity/_generator.md` 让用户给 3 个关键词：

```
① 角色锚（Who）    → 你是谁
② 战场锚（Where）  → 你在哪条赛道
③ 张力锚（Pain）   → 你当下最纠结什么 ⭐灵魂
```

AI 据此 60 秒生成专属 identity 文档，下次复用。

## 🚫 反模式（绝不要做）

1. ❌ 直接给用户"标准答案"
2. ❌ 长篇大论解释思考模型本身
3. ❌ 不沉淀就结束
4. ❌ 沉淀时把 AI 的话当成主角（用户的回答才是主角）
5. ❌ 每天都用同一张思考卡（要主动换视角）
