# 动态身份生成器（_generator）

> 当用户身份不在预设清单（火山引擎 / 数智平台 / 个人 / 创业者 / 解决方案专家）中，调用本生成器，让用户给出 3 个关键词，2 分钟内生成 identity 文档并写入 `identity/<slug>.md`。

---

## 调用时机

用户在 Step 1 选择身份时说：
- "我是 AI 创业者"
- "我是艺术家 / 设计师"
- "我是产品经理 / 投资人 / 学生 / 律师"
- 任何不在预设清单的身份

---

## 三关键词约束（必须主动引导）

不要让用户随便给 3 个词。**结构化提问**：

```
要 60 秒生成专属于你的思考身份卡。请按这个结构给我 3 个词：

① 角色锚（你是谁）：例 "AI 创业者" / "独立插画师" / "B 端产品经理"
② 战场锚（你在哪个赛道）：例 "教育 SaaS" / "AI + 短视频" / "金融科技"
③ 张力锚（你当下最纠结的命题）：例 "找不到 PMF" / "AI 抢饭碗的焦虑" / "增长卡瓶颈"

⚠️ 第三个词最重要——没有张力，思考就是百度百科。
```

如果用户只给前两个，**必须追问第三个**，不要自己脑补。

---

## 4 步推理链

### Step 1：角色映射
基于 ① 角色锚，输出：
- 这个角色在生态中的位置（上游 / 下游 / 中介）
- 典型一天的 3 个关键决策
- 这个角色的"价值产出物"是什么（代码？作品？决策？关系？）

### Step 2：战场扫描
基于 ② 战场锚，输出：
- 这个赛道当下 1 个结构性矛盾（不是表面问题）
- 3 个关键玩家或对标
- 1 个最值得关注的变量（技术 / 政策 / 资本 / 用户行为）

### Step 3：张力放大（核心）
基于 ③ 张力锚，生成 **5-7 个对抗性问题**。质量标准：
- ❌ "你怎么看 PMF" → 太泛
- ✅ "如果你下个月就要关停一条产品线，你会先砍哪条？为什么不是另一条？"
- ✅ "你说的'用户喜欢'，是付费用户喜欢，还是白嫖用户喜欢？"
- 每个问题必须有立场、能逼出选择、不能被"都重要"糊弄过去

### Step 4：信息源推断
基于 ①+②，推断这个身份**真实订阅**的：
- 2-3 个行业媒体
- 3-5 个 X / 推特账号
- 1-2 个社群 / Newsletter

## 写入与复用规则

1. 生成完成后，**写入** `identity/<slug>.md`
2. 在 `MEMORY.md` 的【活跃项目】区登记一条：
   `slow_thinking_identity_<slug>: {{身份名}}, 创建于 YYYY-MM-DD`
3. 下次用户提到同样的身份，**直接读文件**，不再走生成流程
4. 用户可以说"更新我的 [身份] 身份卡"来重新生成

---

## 质量校验（生成后自查）

生成完毕后，AI 自己 review 一遍，必须满足：
- [ ] 对抗性问题没有"你怎么看 / 你觉得如何"这种泛问
- [ ] 至少 1 个问题涉及"取舍 / 不可兼得"
- [ ] 信息源不能凭空编造，不确定时说"建议你补充"
- [ ] 字数控制在 600 字以内，不要长篇大论
